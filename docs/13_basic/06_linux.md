## 리눅스?

리누스 토발즈의 취미생활로 탄생한 OS이며,\
minux를 좀 더 유용하기 만들기 위해 개발하기 시작하여 전 세계 수많은 개발자들이 참여하여\
현재 여러 종류의 배포판들이 끊임없이 출시되고 있다.

- 리눅스의 특징
  - GPL 라이선스
    - 리눅스 사용에 대한 제약이 없으며, 대신 기술지원/유지보수가 필요하면 관련 비용은 발생한다.
    - GPL 라이선스로 자신의 입맛에 맞게 소스를 수정하면, 그에따른 소스 코드 공개가 필요하다.
  - Unix Like
    - 유닉스와 비슷한 환경, 특징을 가지고 있음
  - vi(m) 편집기
    - UNIX는 vi를 기본 편집기로 사용하는데, 리눅스는 이 기능을 확장한 vim 편집기를 기본으로 지원함.
  - 파일 시스템
    - C, D, E 같은 드라이브를 기준으로 파티션을 제공하는 윈도우즈와 달리 리눅스는 디렉터리를 기준으로 제공한다.
    - 최상위 디렉터리를 '/'를 사용한다.
    - 하나의 파티션은 하나의 디렉터리가 될 수 있고, 이를 하는 기능으로 mount라는 개념을 사용함.
    - 디스크 관련된 파일 디렉터리로 '/dev' 디렉터리를 사용한다.
    - 기본 디렉터리 디바이스 장치
      - /dev/null: 파일은 블랙 홀 파일로, 보내온 문자를 모두 지워버린다.
      - /dev/tty: 프로그램이 실행되고 있는 터미널 윈도우 또는 콘솔
      - /dev/dsp: 사운드 카드의 AU 사운드 파일 실행 인터페이스
      - /dev/fd0: 첫 번째 플로피 드라이버
      - /dev/hda1: IDE 하드 디스크의 첫 번째 파티션
      - /dev/sda1: SCSI, S-ATA 하드 디스크의 첫 번째 파티션
      - 마운드를 위한 명령어: `mount -t ext3 /dev/hda1 /backup`
      - 리눅스는 기본적으로 `ext3` 파일 시스템을 사용한다.

- 사용가능한 shell 종류 출력
  - `cat /etc/shells`
