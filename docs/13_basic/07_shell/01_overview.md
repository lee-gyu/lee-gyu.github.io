## Shell이란?

OS에서 제공하는 명령을 실행하는 프로그램.\
OS가 관리하는 자원(파일, 하드웨어, App) 과의 인터페이스를 제공.

- POSIX란? (Portable Operating System Interface)

서로 다른 OS와 프로그램들을 위한 소프트웨어 표준으로 프로그램 이식성 향상을 위한 표준\
UNIX-like 컴퓨팅 환경을 제공하는 것이 목표.\
bash 쉘은 POSIX의 100% 이식성을 가지는 쉘 프로그램이다.

windows OS는 기본적으로 이러한 POSIX를 온전히 지원하는 쉘이 없어,\
windows 기반 프로그래머는 bash에 익숙치 못함.

## Shell에서의 표시

쉘에서 $ 표시는 일반 유저, #표시는 슈퍼유저(e.g. root)를 의미함.\
특정 쉘의 도움말을 얻기 위해서는 `man` 명령을 사용한다. (e.g. `man mount`)

## Shell에서의 환경 변수

현재 환경에서 설정된 변수를 출력하기 위해서 '$'을 앞에 붙여 사용한다.\
'echo $HOME'을 실행하면 HOME이라는 환경 변수 값을 출력한다.

아래와 같은 기본 환경 변수들이 있다.

- '$$': 현재 shell의 pid 식별자

'env' 명령을 입력하면 현재 환경 변수를 모두 출력한다.\
환경변수는 모두 대문자로 이름을 명명한다.

## 쉘 스크립트 구성 요소

- if, else 조건 분기
- for, while 루프
- grep, awk, cut 텍스트 처리 유틸리티
- w, who, free 등의 바이너리 명령

## 쉘 스크립트 사용 이유

- 유저/파일로부터 입력 받아 데이터 출력
- 동일한 작업 자동화
- 자신만의 OS 파워 도구 작성
- 관리자 작업 커스터마이징
- 리눅스 시스템 모니터링
- 데이터 백업/스냅샷 생성
- 데이터베이스 백업 덤프
- 시스템 리소스 파악 (메모리, CPU, 로그인 유저 파악 등)
- 네트워크 서비스 파악
- OS 보안 정책 정리

## 쉘 스크립트 사용을 피해야하는 경우

- 정렬/해쉬 등의 복잡한 연산 작업
- 플랫폼간 이식성이 필요한 프로그램 작성
- 구조적 프로그래밍이 필요한 애플리케이션 작성
- 복잡한 파일 연산
- GUI 작업
- 시스템 하드웨어 접근
- 소켓 I/O 사용 프로그램 작성

## 쉘의 편집 모드

쉘의 편집모드는 emacs 모드와 vi 모드가 있다. (기본은 emacs)\
아래를 통해 지정이 가능하다.

- emacs 모드의 기본 단축키
  - ctrl+b: 커서 뒤로 이동
  - ctrl+f: 커서 앞으로 이동
  - ctrl+p: 히스토리 이전 명령
  - ctrl+n: 히스토리 다음 명령
  - tab: 매칭되는 파일 검색하여 자동 타이핑

- vi 모드의 단축키
  - ESC: 편집 모드
  - h: 왼쪽 커서 이동
  - l: 오른쪽 커서 이동
  - k: 히스토리 이전 명령
  - j: 히스토리 다음 명령

```bash
# 현재 on/off 여부 확인
shopt -o emacs
# vi로 on 설정 (set)
shopt -os vi
```

## 메시지 출력

- printf

```bash
# 이 변수는 현재 쉘이 종료되면 사라짐.
# 현재 쉘 안에서만 유효한 환경 변수
DATE=2022-01-01
printf "%s\n" "$DATE"
```

- echo

```bash
DATE=2022-10-01
echo $DATE
```

## 현재 쉘 종료

아래 쉘 명령이나 Ctrl + D를 통해 쉘을 종료한다.

```bash
logout
exit
```

## 다중 명령어 사용

- ';': 하나의 라인에서 여러 명령어 호출 시, 구분자로 사용함. (첫 명령이 실패해도 다음 명령 실행됨)

```bash
echo ABC; echo BBB
```

- '&&': 하나의 라인에서 여러 명령 호출. 이전 명령 실패 시, 다음 명령 호출X

```bash
echo ABC && echo BBB
```

- '||': 하나의 라인에서 여러 명령 호출, 이전 명령이 실패해도, 다음 명령이 호출된다.

## 명령 히스토리

쉘은 최근 실행한 명령을 저장해둔다. 방향키 Up, Down이나 히스토리 탐색 단축키를 이용하여\
어떤 히스토리가 있는지 파악할 수 있으며, 쉘 명령을 통해서도 이전 명령을 불러올 수 있다.

- !d: 최근 명령 중, d로 시작한 명령
- !!: 가장 최근 명령을 다시 불러옴
- !-1: 최근 명령 -1번째 불러옴
